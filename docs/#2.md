What goes where (fast map)
	тАв	apps/web/app/api/: all HTTP routes (create campaign, schedule, publish, analytics, events, oauth callbacks).
	тАв	apps/web/app/(dashboard|projects|composer): main views. Projects page renders Kanban and Gantt. Composer plans campaigns.
	тАв	apps/web/lib/: database, queues, automations, vendor clients, preflight checks, brand tokens, timing logic.
	тАв	apps/web/public/: sprite and textures for the graffiti theme.
	тАв	packages/workers/: stateless workers for generation, conversion, publishing, analytics, and scheduling.
	тАв	configs/: shared lint, tsconfig, and Drizzle config.
	тАв	infra/: Docker and local compose if you want Postgres and Redis locally.
	тАв	.github/workflows/: CI and worker deploy. Add your secrets in repo settings.

Workspace scripts (root package.json)

Add something like this so you can boot everything with two commands.

{
  "name": "mythos",
  "private": true,
  "workspaces": ["apps/*", "packages/*", "configs/*"],
  "scripts": {
    "dev:web": "pnpm --filter mythos-web dev",
    "dev:workers": "pnpm --filter mythos-workers run-all",
    "build": "turbo run build",
    "lint": "turbo run lint",
    "typecheck": "turbo run typecheck"
  },
  "devDependencies": {
    "turbo": "2.1.3"
  }
}

Minimal first run
	1.	pnpm install (or npm/yarn)
	2.	Copy .env.example to .env and fill values
	3.	Start DB and Redis via docker compose up -d in infra/ (optional)
	4.	pnpm dev:web
	5.	POST /api/dev/seed once to create a demo project
	6.	In another terminal, pnpm dev:workers
